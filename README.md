## 🚀 About the Project

**AWS S3 Uploader** is a cross-platform desktop application designed to facilitate interaction with Amazon S3 buckets, eliminating the need to constantly interact with the AWS web console. With a meticulously crafted and adaptive design (supporting Light and Dark Modes), this application allows users to **upload, download, move, rename, and delete** local files and folders directly and visually in the cloud.

---

## ✨ Key Features

* **🖥️ Modern and Intuitive Interface:** Adaptive interface built on `CustomTkinter` with Light/Dark modes and a classic explorer design featuring breadcrumb navigation.
* **🌍 Dual Language UI / Modelos Multilingües:** Supports Internationalization (i18n). Switch instantly between **English (EN) and Spanish (ES)** languages via a small icon and toggle in the sidebar. The language logic is dynamically loaded from XML files in the `locales/` directory.
* **🔒 Solid Credential Security:** Instead of exposing credentials, it dynamically links and encrypts them at the system level using `keyring`, keeping sessions active for 90 days. You can log in by linking an original `.csv` file generated by AWS.
* **🚚 Bulk Operations:** Full support for moving or downloading tens or hundreds of files simultaneously to a new location.
* **📂 Drag & Drop Support:** Drag files and folders directly from your computer into the application to automatically start uploading to S3.
* **📈 Threads and Concurrency:** The main UI doesn't freeze during heavy operations (massive downloads and uploads). It uses parallel callbacks to reflect the current status using high-precision progress bars.
* **🛡️ Secure Deletion:** Destructive operations like permanent deletion require double approval and a configurable security PIN code.
* **⚙️ Dynamic Storage Classes:** Easily assign if you want to save files as `STANDARD`, `GLACIER_IR`, `DEEP_ARCHIVE`, or `ONEZONE_IA` using a lightweight selector in the toolbar.

## 📂 Project Structure

The code is organized following principles of modularity and separation of concerns:

```text
AWS-S3-UPLOADER/
│   main.py               # Main entry point
│   requirements.txt      # Project dependencies
│   s3icono.png           # Application icon
│
├── core/                 # Backend Logic
│   ├── auth_manager.py   # Login and Keyring controller
│   ├── i18n.py           # XML Parser & Language State Manager
│   └── s3_manager.py     # Native Boto3 SDK for S3 operations
│
├── locales/              # i18n Language Packs
│   ├── en.xml            # English translations
│   └── es.xml            # Spanish translations
│
├── ui/                   # Frontend logic and interfaces
│   ├── explorer_window.py# Main interface (Dashboard)
│   ├── login_window.py   # Authentication window
│   └── move_modal.py     # Functionality and view for moving S3 items
│
└── utils/                # General Utilities
    └── helpers.py        # Cross-cutting functions (e.g., get downloads folder)
```

## 💻 Installation and Setup

### Prerequisites
Make sure you have **Python 3.14** or higher installed on your machine.

1. Clone this repository:
```bash
git clone https://github.com/eduardo-zepeda-j/s3uploader.git
cd s3uploader
```

2. Create your virtual environment (Recommended):
```bash
python3 -m venv venv
# Activate on Windows:
venv\Scripts\activate
# Activate on Mac/Linux:
source venv/bin/activate
```

3. Install dependencies:
```bash
pip install -r requirements.txt
```

## 🕹️ Usage and Workflow

### Start the application
Simply run the main script:
```bash
python3 main.py
```

### Key Functions

1. **File Management / Direct Interaction:** 
   * You'll see a side panel with your buckets on the left.
   * The main view renders unique icons per file type (`.jpg`, `.mp4`, `.pdf`, and dozens more), extracted in $O(1)$ time to greatly speed up heavy rendering operations.
2. **Language Configuration (ES / EN):**
   * Go to the bottom left corner of the SideBar, choose `en` or `es` on the Segmented Button Switcher to configure the language XML packages dynamically on-the-fly.
3. **Temporary Link Generation:**
   * Clicking the link/lock icon (`🔗`) next to files in the list view will generate a presigned URL and instantly copy it to your clipboard.
4. **Move Modal:**
   * Select multiple items, click "Mover Slc." (Move Selected) and a module will activate allowing you to virtually navigate all possible destinations without refreshing the main window, enabling dynamic object transfer between buckets or folders.

## 🔐 Security and Session Management

Upon first launch, the system will ask for a `.csv` access file generated by IAM in AWS.
The application:
* Excludes PyInstaller build files and temporary caches from Git (`.gitignore` rigorously configured).
* Saves your `access_key` in temporary memory and entrusts your `secret_key` to your computer's Native Keychain Security system (Keychain Access on Mac, Credential Manager on Windows). All managed via `core/auth_manager.py`.

> [!WARNING]
> Remember never to commit the local `config.json` file that the app might create. It keeps track of the last accessed path and your anti-deletion security PIN.
